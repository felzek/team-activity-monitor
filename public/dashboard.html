How the answer gets built

  1. Parse the question into teammate, intent, and timeframe.
  2. Resolve the teammate from workspace aliases before calling any APIs.
  3. Fetch Jira and GitHub independently so partial failures stay visible.
  4. Normalize both sources into one grounded response with caveats.

The summary stays trustworthy because the renderer formats normalized facts instead of inventing activity.
