Security overview

Operational and application security posture

This implementation includes local security controls in code now and AWS-oriented production scaffolding for secrets management, deployment isolation, and observability.

Application controls

Secure cookies, CSRF protection, explicit role checks, rate limiting, and strict browser security headers are enabled in the app path.

Auditability

Sign-in events, invitations, connector changes, workspace settings changes, and query execution are all recorded as audit events.

Secrets posture

Connector metadata stores only secret references. Production deployment is prepared for AWS Secrets Manager instead of database-held tokens.

Production deployment

The repository includes Terraform and GitHub Actions scaffolding for AWS-hosted infrastructure, staged deployments, and OIDC-based cloud access.

Data model

Organization scoping is explicit across memberships, connectors, query runs, audit events, and background jobs.

Known roadmap items

Cognito federation, Redis-backed rate limiting, managed queues, and AWS-native secret retrieval are scaffolded operationally and documented for production rollout.
