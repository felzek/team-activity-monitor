How to configure and use the application

The app now uses a local Ollama model for response generation. Jira and GitHub can still run in recorded-fixture mode, but the answer itself comes from a local open-source model instead of a deterministic formatter.

1. Start Ollama locally

Install Ollama, start the local service, and make sure the API is reachable on http://localhost:11434/api.

2. Pull the default model

Run ollama pull qwen2.5:7b, or choose another local model and point OLLAMA_MODEL at it.

3. Verify the model path

Run npm run llm:check before opening the app. That confirms the model exists and can answer a tiny health prompt.

4. Create an owner account

Register from the homepage. Your first sign-up creates an organization automatically.

5. Configure workspace settings

Use the dashboard settings panel to define the team members and repositories the org should track.

6. Run grounded queries

Ask questions like “What is John working on these days?” and review the saved answer, source status, and caveats in the workspace.
