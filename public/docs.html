How to configure and use the application

The local app runs with fixture mode and local auth by default. Production deployment is scaffolded for AWS-managed infrastructure, but this environment remains runnable without cloud credentials.

1. Create an owner account

Register from the homepage. Your first sign-up creates an organization automatically.

2. Configure workspace settings

Use the dashboard settings panel to define the team members and repositories the org should track.

3. Add connector references

Store Jira and GitHub secret references in the connector forms. Production deployment is designed for AWS Secrets Manager.

4. Invite teammates

Create invitations for members, admins, or support users. Each invite produces an auditable registration link.

5. Run grounded queries

Ask questions like “What is John working on these days?” and review the saved response in workspace history.

6. Review ops signals

Check recent audit events, background jobs, and connector status before using the tool in a demo or customer review.
